## 执行流程
1. 先判断用户是否已提供小说的`章节名`、`章节大纲内容`：
- 若未提供，则主动向用户询问`章节名`和`章节大纲内容`。
- 若已提供，则继续执行后续流程。

2. 检查`./output`目录下是否存在已生成的小说章节文件：
- 若无旧章节文件，则当前章节命名为 `第1章 `+`章节名`，直接进入流程4。
- 若有旧章节文件，则当前章节命名为 `第X章 `+`章节名`，其中 X = 已有章节数量 + 1，然后进入流程3。

3. 【仅执行1次】读取`./output`目录下所有旧章节文件（需拼接完整路径），对每一章内容进行总结，提取剧情要点、人物状态、核心情节，合并整理为一份`章节总结`，用于保持剧情连贯。完成读取与总结后，立即进入流程4。

4. 根据用户提供的`章节大纲内容`生成小说正文；若存在`章节总结`，则结合`章节总结`与`章节大纲内容`共同生成，保证剧情连贯。

5. 将生成完成的小说章节保存至`./output`目录。

## 生成要求
- 内容严格贴合用户提供的`章节大纲内容`。
- 标题严格使用指定的`章节名`。
- 剧情、人设、世界观与`章节总结`保持一致（若存在）。
- 单章字数控制在 **1000–2000 字**。
- 保存格式统一为 **.md 文件**。
- 全文叙事风格保持统一。
- 【强制约束】章节结构严格为：**仅1个一级标题 `# 第X章 章节名` + 纯正文段落**，正文内**禁止使用任何 Markdown 标题**（##/### 等均不允许），所有场景、对话、情节以连续自然段落呈现，不得拆分小标题。
- 【强制约束】不得将章节大纲节点直接作为正文小标题，必须将大纲内容完全融入连贯叙事中。
