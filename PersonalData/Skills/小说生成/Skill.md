## 执行流程
1. 先判断用户是否已提供小说的`章节名`、`章节大纲内容`：
- 若未提供，则主动向用户询问`章节名`和`章节大纲内容`。
- 若已提供，则继续执行后续流程。

2. 检查`./output`目录下是否存在已生成的小说章节文件：
- **执行一次** 获取`./output`下面的文件列表
- **若返回结果为空**：当前章节命名为 `第1章 `+`章节名`，**直接进入流程4**。
- **若返回结果不为空**：当前章节命名为 `第X章 `+`章节名`（X = 已有文件数量 + 1），**立即进入流程3**。

3. 【强制：列举后必须立即读取】
- **前提条件**：流程2的  已返回文件列表
- **执行**：**逐个读取**流程2获取到的所有文件（path使用流程2返回的文件名+`/output`）
- **处理**：读取完成后，建立`小说总结索引表`（索引：章节名，值：剧情要点、人物状态、核心情节），合并为`章节总结列表`
- **【强制约束】**：本流程**必须**读取文件，**禁止**再次调用 列出文件列表
- 完成后**立即进入流程4**

4. 根据用户提供的`章节大纲内容`生成小说正文；若存在`章节总结列表`，则结合`章节总结列表`与`章节大纲内容`共同生成，保证剧情连贯。

5. 将生成完成的小说章节保存至`./output`目录。

## 生成要求
- 【强制约束】内容严格贴合用户提供的`章节大纲内容`。
- 标题严格使用指定的`章节名`。
- 剧情、人设、世界观与`章节总结列表`保持一致（若存在）。
- 【强制约束】每个小说总结只能总结一次，禁止重复循环总结。
- 【强制约束】单章节的字数必须在3000字以上，最大不超过5000字。
- 保存格式统一为 **.md 文件**。
- 全文叙事风格保持统一。
- 【强制约束】章节结构严格为：**仅1个一级标题 `# 第X章 章节名` + 纯正文段落**，正文内**禁止使用任何 Markdown 标题**（##/### 等均不允许），所有场景、对话、情节以连续自然段落呈现，不得拆分小标题。
- 【强制约束】不得将章节大纲节点直接作为正文小标题，必须将大纲内容完全融入连贯叙事中。
